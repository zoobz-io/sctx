This document describes sctx's internal design for contributors and advanced users.

### Certificate Verification Scope

> **Important:** sctx does NOT perform full certificate chain verification.

The "Certificate Verification" box in the diagram above performs only:
- Verification that the certificate is signed by a CA in the trusted pool
- Expiration checking

**Full chain verification (root CA trust, intermediate validation, revocation checking) is expected to be performed by your mTLS termination layer** before the certificate reaches sctx.

See [SECURITY.md](../../SECURITY.md#certificate-chain-verification) for deployment requirements.

## Token Generation Flow

When a client requests a token, the following sequence occurs:

```
Client                    Admin Service                    Cache
  │                            │                             │
  │  CreateAssertion()         │                             │
  │  ─────────────────►        │                             │
  │                            │                             │
  │  Generate(cert, assertion) │                             │
  │  ─────────────────────────►│                             │
  │                            │                             │
  │                            │  Verify cert chain          │
  │                            │  ─────────────────►         │
  │                            │                             │
  │                            │  Verify assertion           │
  │                            │  - Check signature          │
  │                            │  - Check fingerprint        │
  │                            │  - Check nonce (replay)     │
  │                            │  - Check expiry             │
  │                            │                             │
  │                            │  Apply policy               │
  │                            │  ─────────────────►         │
  │                            │                Context[M]   │
  │                            │  ◄─────────────────         │
  │                            │                             │
  │                            │  Store context              │
  │                            │  ─────────────────────────► │
  │                            │                             │
  │                            │  Sign token payload         │
  │                            │                             │
  │           SignedToken      │                             │
  │  ◄─────────────────────────│                             │
  │                            │                             │
```

### Assertion Verification

Assertions undergo multiple checks:

1. **Signature verification** - The assertion signature must be valid for the certificate's public key
2. **Fingerprint match** - The assertion fingerprint must match the presented certificate
3. **Nonce uniqueness** - The nonce must not have been used before
4. **Time bounds** - The assertion must not be expired (1 minute default)

### Token Payload

Tokens contain a signed payload:

```go
type tokenPayload struct {
    Fingerprint string    // Certificate fingerprint (cache key)
    IssuedAt    time.Time // When the token was created
    Expiry      time.Time // When the context expires
    Nonce       string    // Unique identifier
}
```

The payload is:
1. JSON-encoded
2. Base64-encoded
3. Signed with the admin's private key

Token format: `base64(payload):base64(signature)`

## Context Cache Design

The cache is keyed by certificate fingerprint (SHA-256 hash of DER-encoded certificate).

```
┌─────────────────────────────────────────────────────────────┐
│                     Memory Context Cache                    │
│                                                             │
│  ┌─────────────────┬────────────────────────────────────┐   │
│  │   Fingerprint   │            Context[M]              │   │
│  ├─────────────────┼────────────────────────────────────┤   │
│  │  abc123...      │  {Perms: [read], Exp: 2024-01-...} │   │
│  │  def456...      │  {Perms: [admin], Exp: 2024-01-..} │   │
│  │  ghi789...      │  {Perms: [write], Exp: 2024-01-..} │   │
│  └─────────────────┴────────────────────────────────────┘   │
│                                                             │
│  Cleanup goroutine (every 5 minutes):                       │
│  - Scan all entries                                         │
│  - Delete expired contexts                                  │
│  - Emit CacheExpired events                                 │
└─────────────────────────────────────────────────────────────┘
```

### Cache Operations

| Operation | Complexity | Notes |
|-----------|------------|-------|
| Get | O(1) | Map lookup |
| Store | O(1) | Map insert |
| Delete | O(1) | Map delete |
| Cleanup | O(n) | Full scan, runs periodically |

### Thread Safety

The cache uses `sync.RWMutex`:
- Read operations (Get) use read lock
- Write operations (Store, Delete) use write lock
- Cleanup acquires write lock only when deleting

## Event System

sctx emits capitan events for all significant operations:

```
Token Lifecycle          Guard Lifecycle          Cache Operations
─────────────────        ────────────────         ─────────────────
TokenGenerated           GuardCreated             CacheStored
TokenVerified            GuardValidated           CacheHit
TokenRejected            GuardRejected            CacheMiss
                                                  CacheDeleted
                                                  CacheExpired

Context Lifecycle        Assertion Lifecycle      Certificate
──────────────────       ────────────────────     ──────────────
ContextRevoked           AssertionValidated       CertificateRejected
                         AssertionRejected
```

Each event includes relevant fields:

```go
// Token generated event
capitan.Emit(ctx, TokenGenerated,
    FingerprintKey.Field(fingerprint),
    CommonNameKey.Field(cert.Subject.CommonName),
    PermissionsKey.Field(strings.Join(perms, ",")),
)
```

## Singleton Enforcement

The admin service is a singleton to ensure:
- Single source of truth for tokens
- Consistent signing keys
- Unified cache

Implementation uses `sync.Once`:

```go
var (
    adminOnce    sync.Once
    adminCreated bool
)

func NewAdminService[M any](key crypto.PrivateKey, cas *x509.CertPool) (Admin[M], error) {
    var result Admin[M]
    var createErr error

    adminOnce.Do(func() {
        result, createErr = createAdminService[M](key, cas)
        if createErr == nil {
            adminCreated = true
        }
    })

    if adminCreated && result == nil {
        return nil, ErrAdminAlreadyCreated
    }

    return result, createErr
}
```

## Nonce Tracking

Nonces prevent assertion replay attacks:

```
┌─────────────────────────────────────────────────────────┐
│                    Nonce Cache                          │
│                                                         │
│  ┌─────────────────────┬──────────────────────┐         │
│  │       Nonce         │     Expiry Time      │         │
│  ├─────────────────────┼──────────────────────┤         │
│  │  abc123...          │  2024-01-15 10:31:00 │         │
│  │  def456...          │  2024-01-15 10:32:00 │         │
│  └─────────────────────┴──────────────────────┘         │
│                                                         │
│  Cleanup: Expired nonces removed on write operations    │
└─────────────────────────────────────────────────────────┘
```

Nonces are stored with their expiry time. Cleanup occurs lazily during write operations (Generate, CreateGuard).

## Cryptographic Design

### Key Detection

The algorithm is detected from the private key type:

```go
func DetectAlgorithmFromPrivateKey(key crypto.PrivateKey) (CryptoAlgorithm, error) {
    switch key.(type) {
    case ed25519.PrivateKey:
        return CryptoEd25519, nil
    case *ecdsa.PrivateKey:
        return CryptoECDSAP256, nil
    default:
        return "", ErrInvalidKey
    }
}
```

### Signing

| Algorithm | Hash | Notes |
|-----------|------|-------|
| Ed25519 | None (built-in) | Signs data directly |
| ECDSA P-256 | SHA-256 | Hashes first, then signs |

### Fingerprinting

Certificate fingerprints use SHA-256:

```go
func GetFingerprint(cert *x509.Certificate) string {
    hash := sha256.Sum256(cert.Raw)
    return base64.StdEncoding.EncodeToString(hash[:])
}
```
