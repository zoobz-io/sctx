# Core Concepts

sctx has five primitives: contexts, tokens, guards, assertions, and the admin service. Understanding these unlocks the full API.

## Admin Service

The admin service is the security authority. It creates tokens, manages contexts, and produces guards.

```go
admin, err := sctx.NewAdminService[UserMeta](privateKey, trustedCAs)
```

The first argument is the signing key for tokens. The second is a certificate pool of trusted CAs.

**The admin is a singleton.** Only one admin service can exist per application. Attempting to create a second returns `ErrAdminAlreadyCreated`.

```go
// First call succeeds
admin1, _ := sctx.NewAdminService[any](key, cas)

// Second call fails
admin2, err := sctx.NewAdminService[any](key, cas)
// err == ErrAdminAlreadyCreated
```

Use `ResetAdminForTesting()` in tests to reset the singleton state. This function requires the `testing` build tag:

```bash
go test -tags=testing ./...
```

## Contexts

A context is the security information extracted from a certificate. It contains permissions, metadata, and expiration.

```go
type Context[M any] struct {
    IssuedAt               time.Time
    ExpiresAt              time.Time
    Metadata               M
    CertificateInfo        CertificateInfo
    CertificateFingerprint string
    Permissions            []string
}
```

Contexts are generic over a metadata type `M`. Use this for application-specific data:

```go
type UserMeta struct {
    Role     string
    TenantID string
}

ctx := &sctx.Context[UserMeta]{
    Metadata: UserMeta{
        Role:     "admin",
        TenantID: "tenant-123",
    },
    Permissions: []string{"users:read", "users:write"},
    ExpiresAt:   time.Now().Add(time.Hour),
}
```

### Context Methods

```go
ctx.HasPermission("read")  // Check for a permission
ctx.IsExpired()            // Check if expired
ctx.Clone()                // Deep copy for parallel processing
```

### Certificate Info

Certificate fields are extracted into a serializable struct:

```go
type CertificateInfo struct {
    CommonName         string
    Organization       []string
    OrganizationalUnit []string
    Country            string
    Province           string
    Locality           string
    SerialNumber       string
    NotBefore          time.Time
    NotAfter           time.Time
    Issuer             string
    IssuerOrganization []string
    KeyUsage           []string
    DNSNames           []string
    EmailAddresses     []string
    URIs               []string
    IPAddresses        []string
}
```

## Tokens

A `SignedToken` is an opaque, tamper-proof reference to a cached context.

```go
token, err := admin.Generate(ctx, clientCert, assertion)
```

Tokens are:
- **Signed** - Cryptographically signed by the admin's private key
- **Verifiable** - Can be verified without accessing the cache
- **Short-lived** - Contain the context expiration time

Tokens are just strings. Pass them in headers, store them in cookies, or include them in request bodies.

## Assertions

An assertion is proof that a client possesses the private key for their certificate. Assertions prevent token theft - even if someone intercepts a token request, they can't generate tokens without the private key.

```go
assertion, err := sctx.CreateAssertion(privateKey, cert)
```

Assertions contain:

```go
type AssertionClaims struct {
    IssuedAt    time.Time  // When the assertion was created
    ExpiresAt   time.Time  // When it expires (1 minute by default)
    Nonce       string     // Random value for replay protection
    Purpose     string     // Always "token-generation"
    Fingerprint string     // Must match the certificate
}
```

Assertions are short-lived (1 minute) and single-use. The admin tracks nonces to prevent replay attacks.

## Guards

A guard enforces permission requirements. Guards are created from tokens and check that the token's context has the required permissions.

```go
guard, err := admin.CreateGuard(ctx, token, "read", "write")
```

Guards are:
- **Bound to a token** - Created from a specific token
- **Permission-specific** - Check for required permissions
- **Reusable** - Can validate multiple tokens with the same permissions

### Guard Validation

```go
err := guard.Validate(ctx, token)
```

Validation succeeds if:
1. The token signature is valid
2. The context hasn't expired
3. The context hasn't been revoked
4. The context has all required permissions

### Guard Methods

```go
guard.ID()          // Unique identifier
guard.Permissions() // Required permissions (copy)
guard.Validate(ctx, tokens...)  // Validate one or more tokens
```

## Policies

A policy defines how certificates become contexts. Set a policy on the admin service:

```go
admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[UserMeta], error) {
    // Extract permissions from certificate fields
    var perms []string
    for _, ou := range cert.Subject.OrganizationalUnit {
        switch ou {
        case "Engineering":
            perms = append(perms, "code:read", "code:write")
        case "Operations":
            perms = append(perms, "infra:read", "infra:write")
        }
    }

    return &sctx.Context[UserMeta]{
        Permissions: perms,
        ExpiresAt:   time.Now().Add(time.Hour),
        Metadata: UserMeta{
            Role:     cert.Subject.CommonName,
            TenantID: cert.Subject.Organization[0],
        },
    }, nil
})
```

Policies can:
- Reject certificates by returning an error
- Grant permissions based on certificate fields
- Set expiration times
- Populate metadata

See [Policy Guide](../3.guides/1.policy.md) for advanced patterns.

## Cryptographic Algorithms

sctx supports two algorithms:

| Algorithm | Constant | Use Case |
|-----------|----------|----------|
| Ed25519 | `CryptoEd25519` | Default, 30% faster |
| ECDSA P-256 | `CryptoECDSAP256` | FIPS 140-2 compliance |

Generate key pairs:

```go
pubKey, privKey, err := sctx.GenerateKeyPair(sctx.CryptoEd25519)
pubKey, privKey, err := sctx.GenerateKeyPair(sctx.CryptoECDSAP256)
```

The algorithm is detected automatically from the key type when creating the admin service.

## Error Types

```go
var (
    ErrInvalidKey          // Invalid private key provided
    ErrAdminAlreadyCreated // Admin singleton already exists
    ErrNoPolicy            // No context policy configured
    ErrInvalidSignature    // Token signature verification failed
    ErrExpiredContext      // Context has expired
    ErrInvalidContext      // Invalid context format
)
```

## Context Cache

Contexts are cached by certificate fingerprint. The default in-memory cache:
- Cleans up expired contexts every 5 minutes
- Uses SHA-256 fingerprints for lookup
- Is thread-safe for concurrent access

Replace the cache implementation:

```go
admin.SetCache(myCustomCache)
```

Custom caches must implement `ContextCache[M]`:

```go
type ContextCache[M any] interface {
    Get(ctx context.Context, fingerprint string) (*Context[M], bool)
    Store(ctx context.Context, fingerprint string, sctx *Context[M])
    Delete(ctx context.Context, fingerprint string) error
    Start(shutdown chan struct{}, wg *sync.WaitGroup)
}
```
