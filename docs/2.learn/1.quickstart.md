# Quickstart

## Requirements

Go 1.23 or later.

## Installation

```bash
go get github.com/zoobzio/sctx
```

## Basic Usage

```go
package main

import (
    "context"
    "crypto/x509"
    "fmt"
    "time"

    "github.com/zoobzio/sctx"
)

func main() {
    // Generate a key pair (or use your existing PKI)
    _, privateKey, _ := sctx.GenerateKeyPair(sctx.CryptoEd25519)

    // Load your trusted CA certificates
    certPool := x509.NewCertPool()
    // certPool.AddCert(yourRootCA)

    // Create the admin service
    admin, _ := sctx.NewAdminService[any](privateKey, certPool)

    // Set a policy that transforms certificates into contexts
    admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[any], error) {
        return &sctx.Context[any]{
            Permissions: []string{"read", "write"},
            ExpiresAt:   time.Now().Add(time.Hour),
        }, nil
    })

    // Client creates an assertion proving key possession
    // (clientKey and clientCert would come from your mTLS setup)
    assertion, _ := sctx.CreateAssertion(clientKey, clientCert)

    // Admin generates a signed token
    ctx := context.Background()
    token, _ := admin.Generate(ctx, clientCert, assertion)

    // Create a guard for authorization
    guard, _ := admin.CreateGuard(ctx, token, "read")

    // Validate the token
    if err := guard.Validate(ctx, token); err != nil {
        fmt.Println("Access denied:", err)
        return
    }

    fmt.Println("Access granted!")
}
```

## What's Happening

1. `NewAdminService` creates the singleton admin that manages tokens
2. `SetPolicy` defines how certificates become security contexts
3. `CreateAssertion` proves the client possesses the private key for their certificate
4. `Generate` verifies the certificate and assertion, then creates a signed token
5. `CreateGuard` creates a permission checker bound to the token
6. `Validate` checks if the token has the required permissions
