# Testing

This guide covers testing patterns for applications using sctx.

## Build Tag Requirement

Testing utilities require the `testing` build tag. Run tests with:

```bash
go test -tags=testing ./...
```

This build tag restriction is a security measure. Without it, `ResetAdminForTesting()` would be available in production builds, allowing malicious code to reset the admin singleton and replace it with attacker-controlled keys.

## Resetting the Singleton

The admin service is a singleton. Reset it between tests:

```go
func TestMyFeature(t *testing.T) {
    sctx.ResetAdminForTesting()

    // Create fresh admin for this test
    admin, err := sctx.NewAdminService[any](privateKey, certPool)
    // ...
}
```

Always call `ResetAdminForTesting()` at the start of each test that creates an admin.

## Generating Test Certificates

Use the testing utilities in `github.com/zoobzio/sctx/testing`:

```go
import sctxtesting "github.com/zoobzio/sctx/testing"

func TestWithCertificates(t *testing.T) {
    // Generate a complete PKI
    testCerts, err := sctxtesting.GenerateTestCertificates()
    if err != nil {
        t.Fatal(err)
    }

    // testCerts contains:
    // - RootCA: Root CA certificate
    // - RootCAKey: Root CA private key
    // - ClientCert: Client certificate signed by root
    // - ClientKey: Client private key
    // - CertPool: Pool containing root CA

    sctx.ResetAdminForTesting()
    admin, _ := sctx.NewAdminService[any](testCerts.ClientKey, testCerts.CertPool)
    // ...
}
```

### Additional Client Certificates

Generate more clients from the same CA:

```go
additionalCert, additionalKey, err := sctxtesting.GenerateAdditionalClientCert(
    testCerts,
    "additional-client",
)
```

### Custom Certificate Attributes

Use `CertBuilder` for certificates with specific attributes:

```go
builder := sctxtesting.NewCertBuilder().
    WithCN("test-service").
    WithOrganization("Acme Corp").
    WithOrgUnit("Engineering", "Security").
    WithValidity(24 * time.Hour)

cert, key, err := builder.SignedBy(testCerts.RootCA, testCerts.RootCAKey)
```

## Testing Policies

Test that your policy correctly transforms certificates:

```go
func TestMyPolicy(t *testing.T) {
    testCerts, _ := sctxtesting.GenerateTestCertificates()

    policy := func(cert *x509.Certificate) (*sctx.Context[UserMeta], error) {
        return &sctx.Context[UserMeta]{
            Permissions: extractPermissions(cert),
            ExpiresAt:   time.Now().Add(time.Hour),
            Metadata:    UserMeta{Role: cert.Subject.CommonName},
        }, nil
    }

    ctx, err := policy(testCerts.ClientCert)

    require.NoError(t, err)
    assert.Contains(t, ctx.Permissions, "expected:permission")
    assert.Equal(t, "test-client", ctx.Metadata.Role)
}
```

### Testing Policy Rejections

```go
func TestPolicyRejectsInvalidCert(t *testing.T) {
    // Create cert without required attributes
    builder := sctxtesting.NewCertBuilder().
        WithCN("unauthorized")
        // Missing required OU

    testCerts, _ := sctxtesting.GenerateTestCertificates()
    cert, _, _ := builder.SignedBy(testCerts.RootCA, testCerts.RootCAKey)

    _, err := myPolicy(cert)

    assert.Error(t, err)
    assert.Contains(t, err.Error(), "missing required")
}
```

## Testing Guards

Test guard creation and validation:

```go
func TestGuardValidation(t *testing.T) {
    testCerts, _ := sctxtesting.GenerateTestCertificates()

    sctx.ResetAdminForTesting()
    admin, _ := sctx.NewAdminService[any](testCerts.ClientKey, testCerts.CertPool)

    admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[any], error) {
        return &sctx.Context[any]{
            Permissions: []string{"read", "write"},
            ExpiresAt:   time.Now().Add(time.Hour),
        }, nil
    })

    // Generate token
    assertion, _ := sctx.CreateAssertion(testCerts.ClientKey, testCerts.ClientCert)
    token, _ := admin.Generate(context.Background(), testCerts.ClientCert, assertion)

    t.Run("valid permissions", func(t *testing.T) {
        guard, err := admin.CreateGuard(context.Background(), token, "read")
        require.NoError(t, err)

        err = guard.Validate(context.Background(), token)
        assert.NoError(t, err)
    })

    t.Run("insufficient permissions", func(t *testing.T) {
        _, err := admin.CreateGuard(context.Background(), token, "admin")
        assert.Error(t, err)
    })
}
```

## Testing Revocation

```go
func TestRevocation(t *testing.T) {
    testCerts, _ := sctxtesting.GenerateTestCertificates()

    sctx.ResetAdminForTesting()
    admin, _ := sctx.NewAdminService[any](testCerts.ClientKey, testCerts.CertPool)

    admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[any], error) {
        return &sctx.Context[any]{
            Permissions: []string{"read"},
            ExpiresAt:   time.Now().Add(time.Hour),
        }, nil
    })

    ctx := context.Background()

    // Generate token and guard
    assertion, _ := sctx.CreateAssertion(testCerts.ClientKey, testCerts.ClientCert)
    token, _ := admin.Generate(ctx, testCerts.ClientCert, assertion)
    guard, _ := admin.CreateGuard(ctx, token, "read")

    // Should work before revocation
    err := guard.Validate(ctx, token)
    require.NoError(t, err)

    // Revoke
    fingerprint := sctx.GetFingerprint(testCerts.ClientCert)
    err = admin.RevokeByFingerprint(ctx, fingerprint)
    require.NoError(t, err)

    // Should fail after revocation
    err = guard.Validate(ctx, token)
    assert.Error(t, err)
}
```

## Capturing Events

Use capitan hooks to verify events are emitted:

```go
func TestTokenGeneratedEvent(t *testing.T) {
    var eventReceived bool
    var capturedFingerprint string

    listener := capitan.Hook(sctx.TokenGenerated, func(ctx context.Context, e *capitan.Event) {
        eventReceived = true
        capturedFingerprint, _ = sctx.FingerprintKey.From(e)
    })
    defer listener.Close()

    testCerts, _ := sctxtesting.GenerateTestCertificates()
    sctx.ResetAdminForTesting()
    admin, _ := sctx.NewAdminService[any](testCerts.ClientKey, testCerts.CertPool)

    assertion, _ := sctx.CreateAssertion(testCerts.ClientKey, testCerts.ClientCert)
    _, _ = admin.Generate(context.Background(), testCerts.ClientCert, assertion)

    // Drain capitan events
    capitan.Shutdown()

    assert.True(t, eventReceived)
    assert.NotEmpty(t, capturedFingerprint)
}
```

## Testing with Multiple Clients

```go
func TestMultipleClients(t *testing.T) {
    testCerts, _ := sctxtesting.GenerateTestCertificates()

    sctx.ResetAdminForTesting()
    admin, _ := sctx.NewAdminService[any](testCerts.ClientKey, testCerts.CertPool)

    admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[any], error) {
        return &sctx.Context[any]{
            Permissions: []string{"read"},
            ExpiresAt:   time.Now().Add(time.Hour),
        }, nil
    })

    ctx := context.Background()

    // Create multiple clients
    client1Cert, client1Key, _ := sctxtesting.GenerateAdditionalClientCert(testCerts, "client-1")
    client2Cert, client2Key, _ := sctxtesting.GenerateAdditionalClientCert(testCerts, "client-2")

    // Generate tokens for each
    assertion1, _ := sctx.CreateAssertion(client1Key, client1Cert)
    token1, _ := admin.Generate(ctx, client1Cert, assertion1)

    assertion2, _ := sctx.CreateAssertion(client2Key, client2Cert)
    token2, _ := admin.Generate(ctx, client2Cert, assertion2)

    // Both should be valid
    guard1, _ := admin.CreateGuard(ctx, token1, "read")
    guard2, _ := admin.CreateGuard(ctx, token2, "read")

    assert.NoError(t, guard1.Validate(ctx, token1))
    assert.NoError(t, guard2.Validate(ctx, token2))

    // Revoking one shouldn't affect the other
    admin.RevokeByFingerprint(ctx, sctx.GetFingerprint(client1Cert))

    assert.Error(t, guard1.Validate(ctx, token1))
    assert.NoError(t, guard2.Validate(ctx, token2))
}
```

## Benchmarking

Run performance benchmarks:

```bash
go test -bench=. -benchmem ./testing/benchmarks/
```

The benchmarks package includes:
- Token generation
- Token verification
- Guard creation
- Guard validation
- Cache operations
- Cryptographic operations

See `testing/benchmarks/core_performance_test.go` for benchmark implementations.

## Integration Testing

For integration tests with real PKI infrastructure, use the integration test suite:

```bash
go test -tags=integration ./testing/integration/...
```

Integration tests cover:
- mTLS handshake flows
- Certificate rotation
- Context revocation
- Multi-CA trust chains

See `testing/integration/README.md` for setup instructions.
