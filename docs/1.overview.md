# Overview

Authorization in Go typically means choosing between JWT complexity or rolling your own token system.

sctx offers a third path: certificate-based security contexts that integrate with your existing PKI.

```go
// Create the admin service (singleton)
admin, _ := sctx.NewAdminService[UserMeta](privateKey, trustedCAs)

// Define your authorization policy
admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[UserMeta], error) {
    return &sctx.Context[UserMeta]{
        Permissions: []string{"read", "write"},
        ExpiresAt:   time.Now().Add(time.Hour),
        Metadata:    UserMeta{Role: "admin"},
    }, nil
})

// Client proves key possession
assertion, _ := sctx.CreateAssertion(clientKey, clientCert)

// Admin generates a signed token
token, _ := admin.Generate(ctx, clientCert, assertion)

// Create guards for authorization checks
guard, _ := admin.CreateGuard(ctx, token, "read", "write")

// Validate tokens against guards
err := guard.Validate(ctx, token)
```

Type-safe, built on mTLS, observable via capitan.

The Admin service is a singleton that manages token generation and guard creation. Contexts are cached by certificate fingerprint. All operations emit capitan events for observability.

## Philosophy

sctx builds on mutual TLS. If you're already doing mTLS, you have certificates. sctx turns those certificates into typed security contexts with permissions, metadata, and expiration.

```go
// Your mTLS handler already has the certificate
func handler(w http.ResponseWriter, r *http.Request) {
    cert := r.TLS.PeerCertificates[0]

    // Turn it into a security context
    assertion, _ := sctx.CreateAssertion(clientKey, cert)
    token, _ := admin.Generate(r.Context(), cert, assertion)

    // Use guards for authorization
    guard, _ := admin.CreateGuard(r.Context(), token, "api:read")
    if err := guard.Validate(r.Context(), token); err != nil {
        http.Error(w, "Forbidden", http.StatusForbidden)
        return
    }
}
```

No separate identity provider. No JWT parsing. Your PKI is your identity system.

## Capabilities

Certificate-based contexts enable:

**[Policy-Driven Authorization](3.guides/1.policy.md)** - Transform certificates into contexts with custom policies. Map organizational units to permissions, extract metadata from certificate fields.

**[Permission Guards](3.guides/2.guards.md)** - Create guards that check for required permissions. Guards can be created by token holders and passed to other services.

**[Revocation](4.cookbook/3.revocation.md)** - Revoke contexts instantly by certificate fingerprint. All guards immediately reject revoked tokens.

**[Observability](3.guides/3.events.md)** - Every operation emits capitan events. Hook into token generation, guard creation, validation successes and failures.

sctx provides the authorization layer. What you build on top is up to you.

## Priorities

### Type Safety

Contexts are generic over a metadata type. No `interface{}` gymnastics.

```go
type UserMeta struct {
    Role      string
    TenantID  string
    Scopes    []string
}

admin, _ := sctx.NewAdminService[UserMeta](key, cas)

admin.SetPolicy(func(cert *x509.Certificate) (*sctx.Context[UserMeta], error) {
    return &sctx.Context[UserMeta]{
        Metadata: UserMeta{
            Role:     extractRole(cert),
            TenantID: cert.Subject.Organization[0],
        },
        Permissions: []string{"read"},
        ExpiresAt:   time.Now().Add(time.Hour),
    }, nil
})
```

### Security

- **Assertion-based authentication** - Clients prove private key possession before token generation
- **Signed tokens** - Tokens are cryptographically signed, tamper-proof references to cached contexts
- **Replay protection** - Nonces prevent assertion replay attacks
- **Certificate verification** - Certificates must chain to trusted CAs

### Performance

- **Context caching** - Contexts are cached by certificate fingerprint
- **Ed25519 by default** - 30% faster than ECDSA, with FIPS-compliant ECDSA P-256 available
- **Lazy cleanup** - Expired contexts are cleaned up periodically in the background
